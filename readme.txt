Итоговые требования

1)Требования к системе 
1.1) Тип операционной системы: Windows 
1.2) Платформа: Microsoft .NET 

2) Функциональные требования 

2.1) Основные функции: 
2.1.2)создавать списки (список создан, на диске файл появляется при первом сохранении)
2.1.3)удалять списки (удаляется файл с физического носителя)
2.1.4)сохранять списки (на диске в текстовом формате .lst)
 Варианты 
- сохранить как: появление диалога сохранения файла как нового.
- сохранить: автоматическое сохранения файла под его именем существующий  файл если файла не существует, вызвать диалог «сохранить как»
2.1.5)открывать списки (только формата .lst)(если список не соответствует не одному из режимов программы, например содержит недопустмые значения, выдать ошибку и предложить открыть другой файл)
2.1.6)удалять элементы списка
 
2.2)Режимы работы: 
Программа должна создавать списки в двух возможных режимах. 
2.2.1) Алфавитный список - верхний и нижний регистр букв, кодировка UTF8(все алфавиты входящие в кодировку UTF8). Ввод не алфавитных символов соопровождается сообщением о недопустимом формате данных. Длинна строки не больше 256 символов.
2.2.2)Цифровой список - целые и числа в диапазоне типа int.
Ввод недопустимых для режима символов должен сопровождаться сообщением о недопустимом формате введенных данных. 
2.3). Управление:
Кнопки: 
1) Delete” - Удаление выбранного элемента списка. Пользователь осуществляет кликом мыши выбор элемента, нажимает на кнопку «Удалить элемент» и элемент удаляется из списка.
2) “Add ” –Добавление элемента в список. Пользователь вводит элемент в текстовое поле и нажимает кнопку «Добавить». Если формат введенного элемента соответствует выбранному режиму, то элемент добавляется в начало списка (первый в столбце). Поле для ввода новых элементов отчищается. 
Горячие клавиши:
1)Ctrl +O - открыть файл из диска тестового формата
2)Ctrl +S - сохранить на носитель список (в текстовом формате)
3)Ctrl +Q - выход из программы
2.4 Поиск и сортировка
2.4.1 поиск в режиме алфавитного списка – нахождение всех элементов, содержащих заданную подстроку, поиск в режиме цифрового списка — поиск заданных чисел/числа в списке.
2.4.2 Сортировка в режиме алфавитного списка — сортировка по буквам в порядке: заглавные буквы, строчные буквы, сортировка в режиме числового списка — сортировка по значению цифр в списке.

2.5 Обработка непредвиденных ситуаций, ошибок. Выход из программы.
Ошибка в программе должна сопровождаться информационным окном, при выходе из программы, должен появится диалог сохранения списка.

3)Нефункциональные требования 
3.1)Устойчивость системы к аварийным сбоям (сохранение введенных пользователем данных в окне приложения с возможностью при новом запуске сохранить информацию в файл) 
3.2)Интерфейс
3.2.1)окно приложения должно появляться посередине экрана и состовлять ¼ от размера экрана
3.2.2)окно должно масштабироваться, без потери внешнего вида элементов на нем. Пояснение: при растяжении окна пропорции элементов на нем должны сохранятся.  
внешний вид окна и шрифты оставить по умолчанию заданные windows forms
